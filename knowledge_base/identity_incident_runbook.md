# Identity Incident Runbook

If a user reports a suspicious login, the service desk must revoke active sessions and force a password reset immediately.

If the user also lost a device, reset MFA only after identity verification is completed and the incident is linked to the support ticket.

The incident record must include the approximate login time, affected application, and whether the source location was expected.

Privileged access cannot be restored until the identity-access team confirms containment and the manager re-approves elevated access.
